Задача: Альтернативный MongoDB клиент
Технологии: Node.js (ES6), MongoDB 3+, Unix OS (Linux, OSX)
Требования к клиенту:
Клиент будет CLI приложением, точно таким же как стандартный mongo клиент (по желаюнию можно сделать HTML UI на AngularJS вместо консольного клиента)
Основной особенность клиента будет то, что запросы в нем будут писаться в виде структурного SQL, вместо стандартного mongo синтаксиса
Достаточно реализовать только выборку(SELECT операцию)
Структура SQL-я может быть следующей

```
SELECT [<Projections>] [FROM <Target>]
    [WHERE <Condition>*]
    [GROUP BY <Field>*]
    [ORDER BY <Fields>* [ASC|DESC] *]
    [SKIP <SkipRecords>]
    [LIMIT <MaxRecords>]
```

GROUP By и аггрегационные функции(count, min, max, avg…) реализовывать не нужно
Для Projections выбираем самый простой вид - *, field, field.subfield, field.*. Подзапросы так же пропускаем.
Condition достаточно сделать для операций - =, <>, >, >=, <, <=. Так же включаем стандартные логические операции - AND, OR, XOR для обьединения Condition-ов
Требований к тому, как печатать результат выборки нету, на свое усмотрение.

Требования к коду:
Понятная Readme.md инструкция, чтобы долго не разбираться с тем, как настроить, сделать build и запускать приложение
Покрытие кода тестами приветствуется
Чистота кода так же приветствуется и будет оцениваться

```
SELECT [<Projections>] [FROM <Target>]
    [WHERE <Condition>*]
    [ORDER BY <Fields>* [ASC|DESC] *]

```
